# Fix 404 Error - Missing /api Path

## Problem
Frontend is calling `/auth/login` instead of `/api/auth/login`

## Quick Fix

### Option 1: Update Code (ALREADY DONE ✅)
The code now automatically adds `/api` to the base URL.

### Option 2: Update Vercel Environment Variable

**If you set VITE_API_URL without /api:**
- Current: `https://voice-dashboard-api.vercel.app`
- Should be: `https://voice-dashboard-api.vercel.app/api`

**To fix in Vercel Dashboard:**
1. Frontend Project → Settings → Environment Variables
2. Find `VITE_API_URL`
3. Edit value to: `https://voice-dashboard-api.vercel.app/api`
4. Redeploy frontend

**But the code fix handles this automatically now!**

## Deploy the Fix

```bash
git add .
git commit -m "Fix API URL to always include /api path"
git push
```

## After Deployment

Test at: `https://voice-dashboard-client.vercel.app`

The login should now work correctly!

## How to Verify

1. **Open** your frontend URL
2. **Open** Browser DevTools (F12) → Network tab
3. **Try to login**
4. **Check** the request URL - should be:
   - ✅ `https://voice-dashboard-api.vercel.app/api/auth/login`
   - ❌ NOT `https://voice-dashboard-api.vercel.app/auth/login`

## If Still 404

Check backend routes in Vercel logs:
1. Backend Project → Deployments → Latest → Functions
2. Should show routes like:
   - `/api/auth/register`
   - `/api/auth/login`
   - `/api/agents`
   - `/api/executions`
